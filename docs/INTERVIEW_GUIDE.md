# Interview Guide

## Explain The System In 60 Seconds

Video Intelligence Agent is a hybrid AI system for CCTV and video analytics. The pipeline first performs local perception with OpenCV, YOLO-based person detection, tracking, optional face identification, and rule-based event extraction. Those results are stored as JSON logs, clips, and summaries. On top of that, I built a query agent that parses user questions, resolves time windows, filters the event log locally, and only then calls Sarvam for higher-level reasoning. That makes the system more practical and production-oriented than an LLM-only demo.

## Why Not Rely Only On Sarvam?

Because raw-video reasoning is expensive, less deterministic, and harder to debug. In this system, the CV pipeline produces structured evidence first. The LLM sees curated event data instead of raw frames, which gives lower cost, better observability, and clearer failure boundaries.

## How Do You Detect Events?

Events are generated by rules over tracks and timestamps. Entry and exit are inferred from movement between image zones, while loitering is based on dwell time plus low movement radius. This gives transparent event logic that is easy to inspect and extend.

## How Does The Pipeline Scale?

The pipeline is modular and stage-based. To scale it, I would replace the offline file reader with per-camera stream readers, maintain tracker state per stream, and push structured events into a queue or database. Because the LLM is not on the frame-by-frame critical path, scaling video ingestion does not require scaling the LLM at the same rate.

## What Makes This Production-Minded?

- local-first perception pipeline
- structured event storage
- evidence clips and debug frames
- fallback behavior when API reasoning is unavailable
- query parsing and filtering before any LLM call
- multiple interfaces: CLI pipeline, Streamlit operator UI, and webcam demo

## Good Follow-Up Improvements

- multi-camera orchestration
- SQLite or PostgreSQL event storage
- FastAPI backend for programmatic access
- richer tracking with DeepSORT or ByteTrack
- optimized CPU / edge deployment profiles
